# Illustration Asset Checklist

Round 13A prepares the final sketch-image production package for externally generated assets. Generate the approved images outside the site, export them as `.webp`, place them in `public/illustrations/final/`, then mark each approved asset as `final` in `src/data/illustrationAssets.ts`.

## Audit Summary

- Asset filenames in `src/data/illustrationAssets.ts` match the target paths below.
- Every asset has English and Chinese alt text in the asset map and in this checklist.
- `GeneratedImagePanel` renders the SVG `SketchMotif` fallback while an asset is `pending`.
- If an asset is marked `final` but the file is missing or fails to load, the component falls back instead of breaking the layout.
- Recommended production sizes below follow the owner brief. Keep the subject centered and crop-safe because several images also appear inside compact cards, drawers, and panel mats.
- No external images, image APIs, scripts, or network requests are required.

## Owner Handoff Steps

1. Open the matching file in `prompts/illustrations/`.
2. Generate the image externally using the English or Chinese prompt.
3. Export as `.webp` at the recommended size.
4. Place the file in `public/illustrations/final/` with the exact filename.
5. Review the image in Home, Shop, Consultation, Story, and Lookbook.
6. When approved, change only that asset's `status` in `src/data/illustrationAssets.ts` from `pendingStatus` to `'final'`.
